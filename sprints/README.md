# Sprints

This folder holds human-facing sprint artifacts:

- sprint demos
- user/stakeholder feedback
- follow-up observations that should shape the next sprint

## Conventions

- Create one markdown file per sprint milestone or demo.
- Name files as `YYYY-MM-DD-<slug>.md`.
- Keep the artifact readable by humans first, with links out to engineering evidence when needed.

## Suggested structure

```markdown
# <Sprint title>

## Demo
What shipped, why it matters, and what was shown live.

## Evidence
Links to implementation, evaluation docs, or logs.

## Feedback
Captured user or stakeholder feedback, with source and date when known.

## Follow-ups
Concrete things to carry into the next engineering step.
```
